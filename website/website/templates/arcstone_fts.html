ARCSTONE Field Test System (FTS) is a GUI application for providing a dynamic quick look of what is occurring during upcoming field tests of the ARCSTONE sprectral instrument at NASA Langley Research Center.

Specfications:

  • OS Developed On: Windows 10
  • Python Version: 3.6
  • 3rd Party modules: numpy, matplotlib, construct, PyQt5, PyInstaller
  • Camera Interface: DigiCamControl
  • One-click executable created using: pyinstaller

Author(s):

  • Jacob Taylor Cassady

1. What is ARCSTONE?

Achieving Instrument High Accuracy In-Orbit or ARCSTONE, is a mission at NASA Langley Research Center (LaRC) in concession with Resonon, Inc., Laboratory for Atomospheric and Space Physics at UC Boulder (LASP), Quartus Engineering, National Institute of Standards and Technology (NIST), and the United States Geological Survey (USGS). Currently in space, there are several devices released by NASA and others which aim to develop an understanding of the Earth's energy budget. Since the Earth gather's almost all of its energy from the Sun, these devices aim to determine the amount of solar irradiance. Although these devices do a great job of understanding the amount of energy hitting the earth, they fail to account for additional sources which shift these values and dampen our ability to accurately determine the amount of energy coming from the sun only. ARCSTONE attempts to rectify this problem by determining the amount of irradiance coming from the next largest source, the moon. Since the lunar irradiance is largely dependent upon the solar irradiance, an apples to apples comparision needs to be made between the Sun and Moon in a short interval. Developing an understanding of this mechanism will allow us to correct all previous, current, and future spectral sensors by using the information from ARCSTONE to develop a lasting model of lunar irradiance as it relates to solar irradiance.

For more information on the ARCSTONE mission please visit: https://arcstone.larc.nasa.gov/

2. Installation of Items Needed For Running and Building ARCSTONE FTS

A list of software requirements for building and running ARCSTONE FTS outside of the one-click executable developed for ease of use.

2.1 Installing Python 3.6

This system is developed entirely in Python. For downloading the specific version of Python (3.6) please visit: https://www.python.org/downloads/release/python-365rc1/. Python versions greater than 3.4 come bundled with pip which is used for many subsequent software installations. If pip is not installed, installation information can be found here: https://pip.pypa.io/en/stable/installing/. Calls for upgrading pip can be found below.

Linux or MacOS

pip install -U pip

Windows

python -m pip install --upgrade pip

2.2 Installing Third-Party Modules and Programs

Installing all 3rd party modules should be done with pip. Additionally, DigiCamControl is needed for communication with additional cameras used during the ARCSTONE field test.

2.2.1 NumPy (1.14.4)

NumPy is the fundamental package for scientific computing with Python. For more information on NumPy please visit: http://www.numpy.org/.

python -m pip install numpy

2.2.2 MatPlotLib (2.2.2)

MatPlotLib is a 2D plotting library for Python. It includes many functions similar to MATLAB. For more information on matplotlib please visit: https://matplotlib.org/.

python -m pip install matplotlib

2.2.3 Construct (2.9.45)

Construct is a parser and builder for binary data. It is used for mapping out the structure of the ScienceEvent binary file. For more information on Construct please visit: https://construct.readthedocs.io/en/latest/.

python -m pip install construct

2.2.4 PyQt5 (5.10.1)

PyQt5 is a Python binding for the Qt application framework. It is used to construct the GUIs for ARCSTONE FTS. For more information on PyQt5 please visit: http://pyqt.sourceforge.net/Docs/PyQt5/. For more information on Qt please visit: https://www.qt.io/.

python -m pip install PyQt5

2.2.5 PyInstaller (3.3.1)

PyInstaller is used to generate the executable for transferring ARCSTONE FTS without the required dependencies. For more information on PyInstaller please visit: https://www.pyinstaller.org/.

Installation:

python -m pip install PyInstaller

Example creation of onefile executable:

"C:\Program Files (x86)\Microsoft Visual Studio\Shared\Python36_64\Scripts\pyinstaller.exe" --onefile --icon=ARCSTONE_icon.ico --clean ARCSTONE_FTS.py

2.2.6 DigiCamControl

DigiCamControl is used to interface with cameras across the command line. This interface is wrapped in ARCSTONE FTS within the Camera.py module. For downloading DigiCamControl as well as documentation please visit: http://digicamcontrol.com/.

3. ARCSTONE FTS Capabilities

ARCSTONE Field Test System focuses on tracking parameters important during field tests as well as providing a display of the binary images produced by ARCSTONE's spectrometers, SWIR and VIS. These field tests are being performed by NASA LaRC and are to take place over the Summer and later this year.

Current Capabilities:

  1. GUI Display
  2. Display of ScienceEvent data
  3. Display of VIS images

3.1 GUI Display

Development of a GUI to quickly and dynamically understand what is happening during the field test improves the chance of a successful test. This is because errors in implementation may be quickly located while in the field and fixed without having to return to our lab to diagnose the logged data. For the development of the GUI, ARCSTONE FTS makes use of a third-party library PyQT5 which acts as a wrapper on the popular C++ GUI framework QT.

3.2 Display of ScienceEvent Data

Tests with the ARCSTONE system are compiled into ScienceEvent files which contain the frames taken and associated information from the test. These ScienceEvent files are binary byte data and are mapped within ARCSTONE FTS using the Construct python module described in 2.2.3 of this document.

3.3 Display of VIS Images

VIS is a spectrometer device developed by the ARCSTONE team using a series of lenses and a camera. Since the VIS and the SWIR are the most important devices for ARCSTONE's gathering of data it is important to make sure our images are being produced from FSW and transmitted to GSE correctly before concluding our test. A quick reference to these images from ARCSTONE FTS would greatly increase our chances of a successful field test. To display these binary images, ARCSTONE FTS makes use of the matplotlib library.
